Externally Available MCP Server

An Interactive Technical Plan & Roadmap

Problem Statement

Our DAM platform must provide an MCP (Media Creation Professional) server so that external AI agents, such as customer chatbots and partner copilots, can securely discover and query DAM-governed resources like help documents, brand assets, and more. This integration must align with our internal Semantic Kernel (SK) framework, enabling SK-based agents to seamlessly call MCP tools.

Business Justification

Strengthen Source of Truth

Extend governed DAM content directly into customer and partner AI ecosystems, reinforcing our platform's central role.

Differentiate from Competitors

Early adoption of MCP servers positions us as a forward-looking leader in enterprise AI integration.

Future-Proof Integrations

MCP provides a standardized "USB-C port" for AI, reducing long-term integration costs and accelerating time-to-market.

Detailed Implementation Plan

This plan outlines a phased, team-based approach to building the MCP server, from a Minimum Viable Product to a full-fledged solution.

MVP (Q1)

DevOps Team

  • Setup initial Azure API Management instance.
  • Configure basic routing to MCP service.
  • Setup Azure AD App Registration for auth.

DAM Backend Team

  • Develop core MCP service host.
  • Implement OIDC token validation.
  • Create static `/discover` endpoint.
  • Create `/invoke` endpoint with synchronous, in-process calls to Semantic Kernel.
  • Implement basic request/response logging.

V1 - Hardening (Q2)

DevOps Team

  • Configure rate limiting in API Management.
  • Setup Azure Service Bus for messaging.
  • Integrate gateway logs with Log Analytics.

DAM Backend Team

  • Refactor `/invoke` to be asynchronous using Azure Service Bus.
  • Create background service (Azure Function) to process queued SK calls.
  • Implement correlation IDs for end-to-end request tracing.

V2 - Full-Fledged (Q3)

DevOps Team

  • Implement advanced security policies in APIM.
  • Setup monitoring and alerting dashboards.

DAM Backend Team

  • Make `/discover` endpoint dynamic.
  • Implement result streaming for long-running tool invocations.
  • Develop comprehensive telemetry system.
  • Implement standardized error handling.

DAM Frontend Team

  • Build admin UI to view MCP usage logs.
  • Connect UI to backend telemetry APIs.

Product Backlog by Version

This backlog is structured by release version, aligning development tasks with the implementation plan.

MVP Backlog (Q1)

Epic: Core MCP Server Infrastructure

  • User Story: As a developer, I want to set up a basic API Gateway to route external requests to the MCP server.
  • User Story: As a developer, I want to implement OIDC token validation to secure the MCP endpoints.

Epic: Tool Discovery and Invocation

  • User Story: As an external agent, I can call a `discover` endpoint to get a static schema of available DAM tools.
  • User Story: As an external agent, I can call an `invoke` endpoint to synchronously execute a tool.
  • Task: Implement basic request/response logging.

V1 - Hardening Backlog (Q2)

Epic: Hardened Infrastructure

  • User Story: As a developer, I want to configure rate limiting and request logging at the gateway level.
  • Task: Set up Azure Service Bus for asynchronous communication.

Epic: Asynchronous Tool Invocation

  • User Story: As a developer, I want to refactor the `invoke` endpoint to publish messages to a service bus for resilience.
  • User Story: As a developer, I want to create a background processor to handle SK invocations from the message queue.
  • Task: Implement correlation IDs for end-to-end request tracing.

V2 - Full-Fledged Backlog (Q3)

Epic: Advanced Governance and Features

  • User Story: As an admin, I want to view detailed logs of all MCP tool invocations, including tenant and arguments.
  • User Story: As an external agent, I want the `/discover` endpoint to be dynamic and always show the latest available tools.
  • User Story: As a developer, I want to implement robust error handling and standardized error responses.
  • Task: Integrate with the central telemetry system to pipe logs and metrics.

Epic: Admin Experience

  • User Story: As an admin, I want a UI inside the DAM to view MCP usage analytics and logs.
  • Task: Build backend APIs to serve telemetry data to the new admin UI.
